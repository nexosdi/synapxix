# MB-101- AI Processing API:
* Design and implement the module in NestJS to receive and process research queries.
  Includes AI client configuration and context management.

This module processes game results using AI with the goal of transforming raw student interaction data into actionable pedagogical insights.

It acts as a bridge between the learning interface and language models, processing the game results already defined by the other team members to then define the child's archetype.


------------------------------------------------------------------------------------

* I am implementing a multimodal system:

Since this allows us to analyze texts and contexts, I found it to be the most scalable approach:

Starting from the idea that each agent adapts better to the pedagogical nature of each activity.

Integrated so far:
Gemini- https://aistudio.google.com/ -- get the API key and paste it into the .env

### Test Curl
The main endpoint is a POST to api/research/process, which can be tested with a curl that must contain:

id, gameType, gameInput (an object with the original game data), studentResult (what the student actually did).

curl -X POST http://localhost:3000/api/research/process

-H "Content-Type: application/json"

-d '{
"studentId": "soperez-2026",
"gameType": "read-aloud",
"gameInput": {
"text": "The quick brown fox jumps over the lazy dog"
},
"studentResult": {
"content": "The quick brown fox jumps over the dog",
"duration": 12,
"success": false
}
}'

### Expected Response
* The server will respond with a JSON including:

* aiFeedback: with a detailed pedagogical analysis generated by Gemini 2.5.

* dimensionUpdate: the impact calculation on dimensions (logic, creativity, etc.).

* performanceSummary: summary of the technical data analyzed.
